# 8. Ghidra와 네이티브 분석

---

# Ghidra MCP가 필요한 경우

- APK 안의 `.so` 라이브러리를 분석해야 할 때
- JNI 함수와 Java bridge를 확인해야 할 때
- 암호화, 문자열 복호화, 안티디버깅 로직을 봐야 할 때
- 의미 없는 함수명을 분석 목적에 맞게 정리해야 할 때

---

# 실무적 가치

- JADX로 보이지 않는 로직을 확인할 수 있다
- Frida 후킹 대상 함수를 선정하는 데 도움이 된다
- 설계상 민감 로직이 클라이언트에 있는지 판단할 수 있다

| 예시 질문 | 기대 결과 |
|---|---|
| JNI 함수가 어떤 데이터를 반환하는가 | 복호화, 조합, 하드코딩 여부 파악 |
| 안티디버깅 흔적이 있는가 | `ptrace`, `TracerPid` 등 후보 정리 |
| 함수명을 바꿔도 되는가 | 리네이밍 제안만 하고 사람 검토 후 반영 |
