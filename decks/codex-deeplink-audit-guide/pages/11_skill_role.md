# Skill이 하는 일

```mermaid
flowchart TB
    A[Skill] --> B[언제 이 흐름을 쓸지 정의]
    A --> C[단계별 실행 순서 정의]
    A --> D[승인 필요 작업 구분]
    A --> E[증거 수집 위치 정의]
    A --> F[Finding 템플릿 정의]
    A --> G[최종 보고서 구조 정의]
```

구체적으로 Skill에 들어가야 할 내용
- 대상 입력값 형식
- static에서 확인할 항목
- dynamic에서 반드시 검증할 항목
- evidence 저장 규칙
- finding 제목과 severity 형식
- MASVS나 내부 기준 매핑 방식

핵심 메시지
- Skill은 단순 설명문이 아니라, Codex가 따라야 할 감사 절차서다.
